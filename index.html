What Is Node.js, and Why Does It Exist?

Node.js is JavaScript running outside a browser. Before Node, JS could only run inside web pages. Node took Chrome's V8 engine and bolted on file access, network calls, and a way to organize code into reusable pieces.

⇄

🍳 Think of it like this

A restaurant. Your code is the order the customer places. Node's APIs are the standard equipment every kitchen has — an oven, a fridge. V8 is the chef who actually reads the ticket and makes decisions. libuv is the kitchen staff who handles the slow stuff — they marinate the meat, wait for deliveries, prep ingredients in parallel. The OS is the outside world — the farm, the supplier, the delivery truck.

Node.js = V8 (runs JS) + libuv (handles everything slow) + a toolbox of APIs on top. Before going deeper into how they work together, let's make this concrete by actually running some code.
